# Glossary

Common terms and definitions used throughout the IWAS documentation.

---

## A

**AAA (Authentication, Authorization, Accounting)**  
Network security protocol used by RADIUS to control access and track usage.

**ARPU (Average Revenue Per User)**  
Average revenue generated per user over a specific period.

**API (Application Programming Interface)**  
Set of protocols and tools for building software applications.

## B

**Bandwidth**  
Maximum rate of data transfer across a network connection.

**Bandwidth Throttling**  
Intentional slowing of internet service by limiting bandwidth.

## C

**Captive Portal**  
Web page displayed to users before granting network access, typically for authentication.

**CSRF (Cross-Site Request Forgery)**  
Security vulnerability where unauthorized commands are transmitted from a user the web application trusts.

**CSAT (Customer Satisfaction Score)**  
Metric measuring customer satisfaction with a product or service.

## D

**Device Binding**  
Associating a WiFi session with a specific device MAC address to prevent sharing.

**DTD (Dart Tooling Daemon)**  
Not applicable to this project (from conversation history context).

## E

**E-Wallet**  
Digital wallet for electronic transactions (e.g., Momo, ZaloPay, VNPay).

**ERD (Entity Relationship Diagram)**  
Visual representation of database schema and relationships.

## F

**FreeRADIUS**  
Open-source RADIUS server for network authentication.

**FR (Functional Requirement)**  
Specific behavior or function that a system must support.

## G

**Guest User**  
User authenticated via Google OAuth without a PC account.

## H

**HMAC (Hash-based Message Authentication Code)**  
Cryptographic hash function used for message authentication.

**Hotspot**  
Physical location offering internet access via WiFi.

## I

**iCafe**  
Internet cafe or gaming center offering PC rental services.

**Idempotency**  
Property ensuring that multiple identical requests have the same effect as a single request.

**IWAS (iCafe WiFi Access Service)**  
The product name for this WiFi monetization platform.

## J

**JWT (JSON Web Token)**  
Compact token format for securely transmitting information between parties.

## K

**KPI (Key Performance Indicator)**  
Measurable value demonstrating how effectively objectives are being achieved.

## L

**Location**  
Physical iCafe site with its own network infrastructure and configuration.

## M

**MAC Address (Media Access Control Address)**  
Unique identifier assigned to network interfaces.

**MikroTik**  
Network equipment manufacturer; RouterOS is their operating system.

**Momo**  
Vietnamese e-wallet and payment platform.

**MVP (Minimum Viable Product)**  
Version of a product with minimum features needed to satisfy early customers.

## N

**NFR (Non-Functional Requirement)**  
System quality attribute (performance, security, scalability, etc.).

## O

**OAuth 2.0**  
Industry-standard protocol for authorization.

**Operator**  
Staff role with view-only access and limited session management capabilities.

## P

**Package**  
WiFi service offering with specific duration, bandwidth, and pricing.

**PC Account**  
User account in the iCafe PC management system.

**Payment Wallet**  
Centralized module for managing multiple payment methods.

**Priority Levels:**

- **P0:** Critical (must-have for MVP)
- **P1:** High (important but not blocking)
- **P2:** Medium (nice-to-have)
- **P3:** Low (future consideration)

## Q

**QoS (Quality of Service)**  
Network traffic management to ensure performance levels.

**QR Code (Quick Response Code)**  
Two-dimensional barcode for storing information.

## R

**RADIUS (Remote Authentication Dial-In User Service)**  
Networking protocol for AAA management.

**RBAC (Role-Based Access Control)**  
Access control method based on user roles and permissions.

**Redis**  
In-memory data structure store used for caching and session management.

## S

**Session**  
Active WiFi connection for a user with specific time and bandwidth limits.

**SLA (Service Level Agreement)**  
Commitment between service provider and customer defining service quality.

**SSO (Single Sign-On)**  
Authentication scheme allowing users to log in once for multiple systems.

## T

**Transaction**  
Payment record for WiFi package purchase.

**Tethering**  
Sharing a device's internet connection with other devices.

## U

**UUID (Universally Unique Identifier)**  
128-bit number used to uniquely identify information.

## V

**VietQR**  
Vietnam's standardized QR code payment system for bank transfers.

**VLAN (Virtual Local Area Network)**  
Logical network segment within a physical network for traffic isolation.

**VNPay**  
Vietnamese payment gateway and e-wallet service.

**VND (Vietnamese Dong)**  
Currency of Vietnam.

## W

**Webhook**  
HTTP callback triggered by specific events, used for real-time notifications.

**WiFi User**  
Customer purchasing WiFi access (can be PC user or guest user).

## Z

**ZaloPay**  
Vietnamese e-wallet and payment platform by Zalo.
